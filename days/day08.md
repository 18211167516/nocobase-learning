## 📖 学习内容

### 1. 工作流概述

#### 什么是工作流？

工作流是 NocoBase 的自动化引擎，可以实现：
- 数据变更时触发操作
- 定时执行任务
- 条件判断和分支
- 发送通知
- 调用外部 API

#### 工作流类型

| 类型 | 说明 |
|------|------|
| 集合事件 | 数据增删改时触发 |
| 定时任务 | 定时执行 |
| 手动触发 | 用户点击按钮触发 |
| Webhook | 外部系统调用触发 |

---

### 2. 创建工作流

#### 步骤 1：创建工作流

1. 进入配置模式
2. 点击「工作流」
3. 点击「添加工作流」
4. 选择触发类型

#### 步骤 2：配置触发器

**集合事件触发器：**
- 选择集合
- 选择触发时机：新增后、更新后、删除后

**定时任务触发器：**
- 设置执行周期
- 设置执行时间

#### 步骤 3：添加节点

从节点库拖拽节点到画布：
- 条件判断
- 数据操作
- HTTP 请求
- 发送通知
- 延时等待

---

### 3. 常用节点

#### 条件判断节点

根据条件执行不同分支：

```
如果 状态 = "已解决"
  → 发送满意度调查
否则
  → 通知处理人
```

#### 数据操作节点

对数据进行增删改查：
- 新增数据
- 更新数据
- 删除数据
- 查询数据

#### HTTP 请求节点

调用外部 API：
- 配置请求地址
- 配置请求方法
- 配置请求参数
- 处理响应结果

#### 通知节点

发送通知：
- 站内通知
- 邮件通知
- 钉钉/企业微信通知

---

### 4. 实战：工单自动分配

#### 场景

当新工单创建时，自动分配给对应分类的负责人。

#### 工作流配置

```
触发器：工单新增后
↓
查询：根据分类查询负责人
↓
更新：设置工单的 assignee 字段
↓
通知：发送通知给负责人
```

---

### 5. 实战：工单状态变更通知

#### 场景

当工单状态变更为"已解决"时，通知创建人。

#### 工作流配置

```
触发器：工单更新后
↓
条件：状态 = "已解决" 且 之前状态 ≠ "已解决"
↓
通知：发送通知给创建人
```

---

### 6. 调试工作流

#### 执行历史

查看工作流的执行记录：
1. 进入工作流配置
2. 点击「执行历史」
3. 查看每次执行的详情

#### 调试技巧

- 使用「测试」功能手动触发
- 检查变量值是否正确
- 检查条件判断逻辑
