## 📖 学习内容

### 1. NocoBase 是什么？

**NocoBase** 是一个**开源的、极易扩展的 AI 无代码开发平台**，通过配置和拖拽来搭建业务系统，而不需要写代码。

它解决了以下问题：
- 团队需要内部系统，但市面软件不合适
- 定制开发太贵、太慢
- 用 Excel 凑合，数据乱、协作难

---

### 2. 五个核心理念

#### ① 数据模型驱动（最重要！）

传统无代码工具是"表单驱动"或"表格驱动"，界面和数据绑定在一起。

NocoBase 采用**数据模型驱动**：
- 先定义数据结构（Collection、Field）
- 再用区块展示数据（表格、表单、图表等）
- 最后用操作处理数据（新增、编辑、删除等）

**优势：** 界面与数据彻底解耦，同一个数据可以创建任意多个不同形式的展示界面。

```
数据源 → 区块 → 操作
(定义数据) (展示数据) (处理数据)
```

#### ② 所见即所得

- 页面就是画布，像搭 Notion 一样直观
- 一键切换配置模式和使用模式
- 点击右上角荧光笔图标进入配置模式

#### ③ 一切皆插件

- 所有功能都是插件，类似 WordPress
- 核心极简（约 200KB），功能按需安装
- 可以自己开发插件扩展功能

#### ④ AI 融入业务

- 内置 AI 员工功能
- AI 可以担任翻译员、分析员、调研员等角色
- AI 与业务流程无缝集成

#### ⑤ 开源 + 私有部署

- 核心代码完全开源（Apache 2.0 协议）
- 数据完全在自己服务器上
- 不按用户/应用/数据量计费

---

### 3. 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React + Ant Design 5.0 |
| 后端 | Node.js + Koa |
| 数据库 | PostgreSQL / MySQL / MariaDB |
| 部署 | Docker / Kubernetes |

**为什么选择 React？**
- Ant Design 组件库成熟，适合企业级应用
- React 生态丰富，适合复杂业务场景
- 灵活性高，适合插件化架构

---

### 4. 两种核心模式

| 模式 | 说明 | 适用人群 |
|------|------|----------|
| **使用模式** | 普通用户日常使用的界面 | 所有人 |
| **配置模式** | 搭建和调整界面的设计模式 | 管理员 |

**切换方式：** 点击右上角的「界面配置（UI Editor）」按钮（荧光笔图标）

在配置模式下，可配置的元素会出现**橙色高亮框**。

---

### 5. 界面基本结构

```
┌──────────────────────────────────────────┐
│ 顶部导航栏 │
├──────────┬───────────────────────────────┤
│ │ │
│ 左侧 │ 内容区域 │
│ 菜单 │ （放置各种区块） │
│（group）│ │
│ │ │
└──────────┴───────────────────────────────┘
```

- **顶部导航栏**：一级菜单，切换不同模块
- **左侧菜单**：二级菜单，组织页面层级
- **内容区域**：页面主体，放置各种区块

---

### 6. 三个关键概念

#### 数据源（Data Source）
- 定义数据结构
- 可以是主数据库、外部数据库或 API
- 是整个系统的核心

#### 区块（Block）
- 在页面上展示数据的方式
- 表格、表单、图表、日历等
- 同一个数据源可以创建任意多个区块

#### 操作（Action）
- 对数据进行处理的操作
- 新增、编辑、查询、导入、导出、触发工作流等

**三步搭建系统：**
```
1. 定义数据源 → 2. 用区块展示 → 3. 用操作处理
```
