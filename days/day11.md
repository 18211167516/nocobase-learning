## 📖 学习内容

### 1. 配置不生效问题

#### 问题 1：环境变量不生效

**症状：** 修改了环境变量，但配置没有变化

**排查步骤：**
```bash
# 1. 检查环境变量是否正确设置
docker compose exec app env | grep DB

# 2. 检查配置文件格式
cat docker-compose.yml

# 3. 重新加载配置
docker compose down
docker compose up -d
```

**常见原因：**
- 环境变量格式错误
- 未重启容器
- 配置文件缩进错误

---

#### 问题 2：插件配置不生效

**症状：** 安装插件后功能不可用

**排查步骤：**
```bash
# 1. 检查插件是否启用
# 进入配置模式 → 插件管理

# 2. 检查插件日志
docker compose logs app | grep plugin

# 3. 重新安装插件
docker compose exec app yarn pm:disable <plugin-name>
docker compose exec app yarn pm:enable <plugin-name>
```

---

### 2. 权限问题

#### 问题 1：用户无法访问菜单

**症状：** 用户登录后看不到某些菜单

**排查步骤：**
1. 检查用户角色
2. 检查角色的菜单权限配置
3. 检查菜单是否启用

**解决方案：**
```
配置模式 → 用户和权限 → 角色 → 菜单权限
勾选需要访问的菜单
```

---

#### 问题 2：用户无法操作数据

**症状：** 用户可以看到数据但无法编辑/删除

**排查步骤：**
1. 检查角色的操作权限
2. 检查数据权限配置
3. 检查字段权限配置

**解决方案：**
```
配置模式 → 用户和权限 → 角色 → 操作权限
为对应集合勾选需要的操作
```

---

#### 问题 3：数据权限过滤不正确

**症状：** 用户能看到不应该看到的数据

**排查步骤：**
1. 检查数据权限条件
2. 检查变量引用是否正确
3. 测试条件表达式

**常见错误：**
```
# 错误写法
created_by = currentUser.id

# 正确写法
created_by = {{currentUser.id}}
```

---

### 3. 数据同步问题

#### 问题 1：外部数据源同步失败

**症状：** 外部数据库的数据无法显示

**排查步骤：**
```bash
# 1. 检查网络连通性
docker compose exec app ping <external-db-host>

# 2. 检查数据库连接
docker compose exec app nc -zv <external-db-host> <port>

# 3. 查看错误日志
docker compose logs app | grep -i "data source"
```

**常见原因：**
- 网络不通
- 防火墙阻止
- 数据库用户权限不足
- 表结构不兼容

---

#### 问题 2：API 数据源请求失败

**症状：** API 数据无法加载

**排查步骤：**
1. 检查 API 地址是否正确
2. 检查认证信息
3. 检查返回数据格式

**调试方法：**
```bash
# 手动测试 API
curl -X GET "https://api.example.com/data" \
  -H "Authorization: Bearer <token>"
```

---

### 4. 性能问题

#### 问题 1：页面加载慢

**可能原因：**
- 数据量大
- 查询未优化
- 服务器资源不足

**解决方案：**
1. 添加分页
2. 优化查询条件
3. 添加索引
4. 增加服务器资源

---

#### 问题 2：工作流执行慢

**可能原因：**
- 节点过多
- 外部 API 响应慢
- 数据量大

**解决方案：**
1. 简化工作流
2. 添加超时设置
3. 使用异步处理
