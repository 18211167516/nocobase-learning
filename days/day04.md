## 📖 学习内容

### 1. Collection（集合）详解

Collection 是 NocoBase 中数据的基本组织单位，类似于数据库中的表。

#### 创建 Collection

1. 进入配置模式
2. 点击「数据源」→「主数据源」
3. 点击「创建集合」
4. 配置集合属性：
   - **集合名称**：数据库表名（英文，如 `tasks`）
   - **集合标题**：显示名称（中文，如「任务」）
   - **继承**：可继承其他集合的字段

#### 系统内置字段

每个 Collection 自动包含：
- `id`：主键
- `created_at`：创建时间
- `created_by`：创建人
- `updated_at`：更新时间
- `updated_by`：更新人

### 2. Field（字段）详解

#### 基础字段类型

| 类型 | 字段名 | 说明 | 配置项 |
|------|--------|------|--------|
| 单行文本 | `string` | 短文本 | 默认值、必填、唯一 |
| 多行文本 | `text` | 长文本 | 默认值、必填 |
| 数字 | `number` | 整数或小数 | 精度、范围、默认值 |
| 布尔 | `boolean` | 是/否 | 默认值 |
| 日期 | `date` | 日期时间 | 格式、默认值 |
| 单选 | `select` | 单个选项 | 选项列表、颜色 |
| 多选 | `multipleSelect` | 多个选项 | 选项列表、颜色 |
| 邮箱 | `email` | 邮箱格式 | 验证规则 |
| 手机号 | `phone` | 手机号格式 | 验证规则 |
| URL | `url` | 网址格式 | 验证规则 |

#### 高级字段类型

| 类型 | 说明 | 使用场景 |
|------|------|----------|
| JSON | JSON 数据 | 存储复杂结构 |
| 密码 | 加密存储 | 用户密码 |
| 算式 | 计算字段 | 自动计算 |
| 序列号 | 自动编号 | 工单编号 |

---

### 3. 字段配置选项

#### 通用配置

- **字段名称**：数据库字段名（英文）
- **字段标题**：显示名称（中文）
- **默认值**：新建记录时的默认值
- **必填**：是否必须填写
- **唯一**：值是否不能重复

#### 验证规则

- **正则表达式**：自定义验证规则
- **最小/最大长度**：文本长度限制
- **最小/最大值**：数字范围限制

### 4. 关联关系

#### 一对多关系（最常用）

**场景：** 一个分类下有多个任务

**配置步骤：**
1. 创建「分类」集合
2. 创建「任务」集合
3. 在「任务」集合中添加「所属分类」字段
4. 字段类型选择「多对一」
5. 选择关联集合为「分类」

```
分类 (1) ←→ (多) 任务
```

#### 多对一关系

**场景：** 多个任务属于一个分类

**配置：** 在「任务」表中添加「分类」字段，类型选择「多对一」

#### 一对一关系

**场景：** 一个用户有一个档案

**配置：** 在「档案」表中添加「用户」字段，类型选择「一对一」
