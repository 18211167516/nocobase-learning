## 📖 学习内容

### 1. 多对多关系

#### 场景

一个任务可以有多个标签，一个标签可以属于多个任务。

```
任务 ←→ 标签
(多)   (多)
```

#### 配置步骤

1. 创建「任务」集合
2. 创建「标签」集合
3. 在「任务」集合中添加「标签」字段
4. 字段类型选择「多对多」
5. 选择关联集合为「标签」
6. NocoBase 会自动创建中间表

#### 中间表

多对多关系会自动创建中间表：
- 表名：`tasks_tags`（自动生成）
- 字段：`task_id`, `tag_id`

---

### 2. 外部数据源

#### 什么是外部数据源？

连接 NocoBase 之外的数据库，直接读取和操作外部数据。

#### 支持的数据库

| 数据库 | 支持情况 |
|--------|----------|
| PostgreSQL | ✅ 完全支持 |
| MySQL | ✅ 完全支持 |
| MariaDB | ✅ 完全支持 |
| Oracle | ✅ 企业版支持 |
| SQL Server | ✅ 企业版支持 |

#### 配置步骤

1. 进入配置模式
2. 点击「数据源」→「添加数据源」
3. 选择数据库类型
4. 配置连接信息：
   - 主机地址
   - 端口
   - 数据库名
   - 用户名/密码
5. 测试连接
6. 保存

#### 注意事项

- 外部数据源只读取表结构，不会修改
- 可以对外部数据进行增删改查
- 不建议修改外部系统的核心表结构

---

### 3. API 数据源

#### 什么是 API 数据源？

通过 REST API 获取外部数据，作为 NocoBase 的数据源。

#### 配置步骤

1. 进入配置模式
2. 点击「数据源」→「添加数据源」
3. 选择「HTTP API」
4. 配置 API 信息：
   - 基础 URL
   - 认证方式（API Key / OAuth / 无）
   - 请求头
5. 配置数据映射

#### 数据映射配置

```json
{
  "list": "$.data.items",
  "total": "$.data.total",
  "id": "$.id",
  "name": "$.name"
}
```

### 4. 字段高级配置

#### 算式字段

自动计算其他字段的值：

```
总价 = 单价 × 数量
```

配置：
1. 添加「算式」字段
2. 输入公式：`{{单价}} * {{数量}}`
3. 设置结果类型（数字）

#### 序列号字段

自动生成编号：

```
工单编号：TK-2026-00001
```

配置：
```
前缀：TK-
日期格式：YYYY
起始值：1
位数：5
```
