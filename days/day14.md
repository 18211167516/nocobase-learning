> **学习目标：** 模拟客户问题、掌握问题排查流程、学习沟通技巧
> **预计时间：** 2 小时
> **难度：** 🔴 重点

## 📖 学习内容

### 1. 售后支持流程

#### 标准流程

```
1. 接收问题
↓
2. 问题分类
↓
3. 问题排查
↓
4. 提供解决方案
↓
5. 验证解决
↓
6. 记录归档
```

#### 问题分类

| 类别 | 说明 | 响应时间 |
|------|------|----------|
| 紧急 | 系统无法使用 | 1 小时内 |
| 高 | 核心功能异常 | 4 小时内 |
| 中 | 功能问题 | 24 小时内 |
| 低 | 咨询类问题 | 48 小时内 |

### 2. 模拟案例

#### 案例 1：无法登录系统

**客户描述：**
> 我输入了正确的账号密码，但是无法登录，一直提示"账号或密码错误"

**排查步骤：**
```markdown
1. 确认问题
   - 账号是什么？
   - 是否刚安装的系统？
   - 之前能否正常登录？

2. 检查账号
   - 检查数据库中用户是否存在
   - 检查密码是否正确
   - 检查账号是否被禁用

3. 检查日志
   - 查看应用日志
   - 查看是否有登录失败记录

4. 解决方案
   - 如果是默认账号，提示修改过密码
   - 如果忘记密码，提供重置方法
```

**回复模板：**
```
您好，感谢反馈。请确认以下信息：

1. 您使用的账号邮箱是什么？
2. 这是新安装的系统吗？
3. 之前是否修改过默认密码？

如果是新安装的系统，默认账号密码是：
- 账号：admin@nocobase.com
- 密码：admin123

如果密码已被修改，可以通过以下方式重置：
[提供重置步骤]
```

#### 案例 2：数据无法保存

**客户描述：**
> 我在表单中填写了数据，点击保存后提示错误，数据无法保存

**排查步骤：**
```markdown
1. 获取错误信息
   - 具体的错误提示是什么？
   - 截图或错误日志

2. 检查字段验证
   - 必填字段是否填写
   - 字段格式是否正确
   - 是否有唯一性冲突

3. 检查权限
   - 用户是否有创建权限
   - 数据权限是否限制

4. 检查日志
   - 查看应用日志
   - 查看数据库日志
```

---

#### 案例 3：页面加载很慢

**客户描述：**
> 系统最近变得很慢，打开页面要等很久

**排查步骤：**
```markdown
1. 确认问题范围
   - 所有页面都慢还是特定页面？
   - 什么时候开始变慢的？
   - 数据量是否有增长？

2. 检查服务器资源
   - CPU 使用率
   - 内存使用率
   - 磁盘空间

3. 检查数据库
   - 慢查询日志
   - 数据量大小
   - 索引情况

4. 检查网络
   - 网络延迟
   - 带宽使用
```

### 3. 沟通技巧

#### 倾听与确认

- 仔细听取客户描述
- 复述确认理解正确
- 询问关键细节

#### 表达清晰

- 使用简单易懂的语言
- 避免技术术语
- 分步骤说明

#### 态度友好

- 保持耐心
- 表示理解
- 积极帮助
