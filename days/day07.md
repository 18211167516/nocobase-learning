## 📖 学习内容

### 1. 用户管理

#### 用户表

NocoBase 内置用户表（`users`），包含：
- 用户名
- 邮箱
- 密码
- 昵称
- 手机号
- 角色

#### 添加用户

1. 进入配置模式
2. 点击「用户和权限」→「用户」
3. 点击「添加用户」
4. 填写用户信息

---

### 2. 角色管理

#### 什么是角色？

角色是一组权限的集合，用于控制用户能访问哪些功能和数据。

#### 内置角色

| 角色 | 说明 |
|------|------|
| Admin | 系统管理员，拥有所有权限 |
| Member | 普通成员，基本权限 |

#### 创建角色

1. 进入配置模式
2. 点击「用户和权限」→「角色」
3. 点击「添加角色」
4. 配置角色名称和权限

---

### 3. 权限类型

#### 菜单权限

控制角色能看到哪些菜单。

配置：
1. 进入角色配置
2. 点击「菜单权限」
3. 勾选可访问的菜单

#### 操作权限

控制角色能执行哪些操作。

| 操作 | 说明 |
|------|------|
| 查看 | 查看数据 |
| 创建 | 新增数据 |
| 编辑 | 修改数据 |
| 删除 | 删除数据 |
| 导出 | 导出数据 |
| 导入 | 导入数据 |

#### 数据权限

控制角色能看到哪些数据。

**数据范围：**
- 全部数据
- 本人的数据
- 本部门的数据
- 自定义条件

**配置示例：**
```
普通员工只能看到自己的工单
条件：created_by = {{currentUser.id}}
```

---

### 4. 权限配置实战

#### 场景：工单系统权限

**角色 1：管理员**
- 菜单权限：全部菜单
- 操作权限：全部操作
- 数据权限：全部数据

**角色 2：客服人员**
- 菜单权限：工单管理、工单列表
- 操作权限：查看、创建、编辑
- 数据权限：分配给自己的工单

**角色 3：普通用户**
- 菜单权限：提交工单、我的工单
- 操作权限：查看、创建
- 数据权限：自己创建的工单

---

### 5. 权限配置步骤

#### 步骤 1：创建角色

```
角色名称：客服人员
角色标识：agent
```

#### 步骤 2：配置菜单权限

勾选可访问的菜单项。

#### 步骤 3：配置操作权限

为每个集合配置操作权限：
- 工单：查看 ✓、创建 ✓、编辑 ✓、删除 ✗

#### 步骤 4：配置数据权限

设置数据过滤条件：
```
assignee = {{currentUser.id}}
```
